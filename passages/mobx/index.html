mobx

因为大多数情况下我们都是结合react来使用mobx，所以我先把实际应用的部分放在前面，具体的在慢慢拆开跟大家慢慢探讨；

结合React使用mobx

在react中，mobx-react提供了 Provider 组件用来包裹最外层组件节点，并且传入 store（通过）context 传递给后代组件：

  • 多个store注入

虽然多个store注入是可行的，但我们通常不会这样使用，如果页面较为复杂需要多个store一起使用，我们就建一个总store文件，在这个store里面去实例化其他store，然后集中给一个组件使用，这样更符合我们的开发习惯

一般情况在实际项目中，mobx都要配合mobx-react一起使用的，那么让我们先来了解一下常用的api有哪些吧：

mobx API

  • @observable

  • @action

    • 任何修改state的操作都称为actions

  • @computed

  • toJS…等

mobx-react API

  • @observer
  • Provider
  • @inject
  • componentWillReact
    • 在添加@observer后，你的组件会多一个生命周期componentWillReact。当组件内被observable观测的数据改变后，就会触发这个生命周期。
    • 注意setState并不会触发这个生命周期！state中的数据和observable数据并不算是一类。
    • 另外被observable观测数据的修改是同步的，不像setState那样是异步，这点给我们带了很大便利。如果想要让setState同步还要使用setState的第二个参数解决。
  • PropTypes…等

细粒度拆分

  1. 当一个observable只被一个组件使用时，将这个变量放在组件中，不要放在store中
  2. 当有多个组件使用一个observable的变量时，有以下几种方式，去传递使用这个变量
    a. 将这个变量放到store中, 引入该store然后使用
    b. 通过props传递到子组件(注意@observer的问题 / 传递原始值和传递对象的问题 / 使用Observer组件的问题)
    c. 跨组件传递可以通过Provider / inject 注入
